§2AI usage — every call, named

CapabilityModelWrapperWhere usedCaching
Tutor answers (Uzbek Socratic) Gemini 2.5 Flash lib/ai/gemini.ts via LanguageModel /api/ask → ask card live (low cost)
Tutor fallback OpenAI gpt-5-mini lib/ai/openai.ts same route, swap by env live
Diagram generation OpenAI gpt-image-1 lib/ai/image.ts /api/generate-image → <DiagramCard> SHA256(prompt+size+quality) → Firestore images/{hash}
Curriculum extraction (vision) OpenAI gpt-5.1 scripts/extract-curriculum.ts offline pipeline Firestore curriculum/{grade}/{subject}/{lessonId}
Health probe Gemini 2.5 Flash adapter GET /api/health —

Pitch line. Every language call flows through one LanguageModel interface (lib/ai/model.ts). Swap Gemini ↔ OpenAI by flipping an env var. The product is not a model wrapper — the agents, the Card DSL, the Curriculum RAG, the emaktab data signal, and the Firestore image cache are ours.
