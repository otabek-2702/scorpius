01
Feedback · Simulations don’t replace the lab

Build instruments, not exhibits.

Muxammadali · verbatim

“xozir o’ziz aytgan shartni o’ziz qanoatlantrimayapsiz labaratoriyaga sharoit yo’q biz shunga qildik deyapsizlar qani labaratoriya silar o’quvchi o’rnida bo’lsanglar shu ishilar qoniqarmidingar dedi.”

What’s true today

11 simulations exist (Brachistochrone, Brownian, Buoyancy, Eclipse, Prism, Lens, Lever, Pascal, Sound, Circuit, DivisorGrid). The physics is real — requestAnimationFrame integration loops, Snell’s law, Archimedes, Ohm. Animations are smooth. But every sim is a single-screen scenario with a hidden win condition: drop 3 blocks → onComplete?.(). Drag past focal point → onComplete?.(). Hear a tone → onComplete?.(). That is structurally a quiz disguised as a sim.

The PhET / Brilliant / Labster bar

A real virtual lab gives the learner a system, not a question. PhET’s Energy Skate Park lets you drag-build any track shape, then watches energy bars animate; Wave Interference drops measurement probes anywhere in the field. Labster lets you set a P200 pipette volume on a virtual physical dial. Algodoo is pure-sandbox 2D physics with a pen tool. None of them say “you finished, here’s a green check.” Completion is the learner’s own aha.

Four criteria all good sims hit: multi-knob continuous control, multiple synchronized representations (visual + numeric + graph), free-roam state space, and no win condition.

Ship this week · 3 builds, in order
1
Kill the win-conditions. Add a Lab Notebook.
Remove every onComplete-triggers-after-N-actions in components/learn/sims/. Replace with a persistent right-side panel where every reading the user takes (τ, V·I, v/u, displaced volume, spectrum spread) appends with a timestamp. Now the sim is an instrument that records — not a quiz dressed up as one.
Effort · 1–2 days · touches all 10 sims
2
Add sandbox mode to the 3 strongest sims.
Buoyancy — 7th shelf item opens density + size sliders before drop. Lever — “+ Add weight” spawning a draggable mass (1–20 kg slider). Circuit — palette of components dragged onto a grid for series/parallel topologies. These three carry the demo: make them sandboxes, not exhibits.
Effort · 3–5 days · touches 3 files

Brownian needs ⟨r²⟩ vs t (the actual physics — diffusion is linear in time, only visible on a graph). Sound needs a spectrum strip below the waveform. Effort · 2 days · reusable component
YC-mentor verdict
The mentor is half-right. The physics is genuine, the metaphors (gold buoyancy arrows, fly-as-object, Brownian zig-zag) are strong. What’s missing is freedom. PhET wins on freedom and tools-you-deploy, not on prettier graphics. Don’t build new sims. Convert the 10 you have from scenario to instrument. That closes the gap and is the difference between “Grade-6 textbook with animations” and “the PhET of Uzbekistan.”

Feedback · The individual approach isn’t shown end-to-end

Make the model visible.

Muxammadali · verbatim

“keyin har bitta o’quvchiga individual yondashib smulyatsiya qilamiz desak shuni xam to’liq qilib ko’rsatinglar dedi.”

What’s actually in the code

Plenty exists. Profile capture — name, grade, interests, sub-interests (Batman/Real Madrid), goal, painPoint — in lib/profile.ts:5-17. Personas inject profileLine() into every LLM system prompt (lib/personas.ts:268-284). The emaktab live snapshot pipeline pulls real grades + diary (lib/emaktab/live.ts). There’s even a weakest-lesson picker (lib/emaktab/weakest.ts) that scans diary marks ≤ 3.

But: pickWeakestLesson at line 37 is hardcoded to return "density" for every user, regardless of which topic the bad mark was on. SkyView.tsx:336 hardcodes const streak = 1. No per-topic mastery state exists. The Batman/Real-Madrid personalization only enters the LLM prompt — the student has no UI hint that “your example uses Batman.” And app/personalizing/page.tsx is a theatrical 5-stage loader (“843 ta baho yuklanmoqda…”) — performed personalization, not measured.

What other EdTech apps actually SHOW

Five UI primitives the leaders use to make personalization observable: (A) per-skill mastery meter — Khan’s mastery bar, IXL’s SmartScore, NWEA RIT. (B) named “next thing” card with a reason — Khanmigo’s “you got stuck on equivalent fractions”, Brilliant’s “Continue” + “Recommended for you.” (C) decay/risk indicator — Duolingo’s crown levels dimming, Anki’s color-coded due cards. (D) visible difficulty fork — IXL stepping up after a streak, Brilliant’s “Quick check — making sure this clicked.” (E) greeting that names prior state — Duolingo home: “Hi Murod · streak 47 · weakest skill: past tense.”

The pattern: every screen names a piece of per-user state and shows it changing. Personalization isn’t a backend fact — it is a visible delta.

The gap · what a 60-second demo needs

To convince a pitch judge, you’d need to point at, on stage, five things we don’t have:

— A per-topic mastery percentage on each star (“EKUB: 60% · 2/5 mashq”).
— A named “Bugungi tavsiya — chunki…” card with a reason sentence.
— A visible difficulty fork inside a lesson (“Bu osonroq bo’ldi, keyingisi qiyinroq” toast).
— A “siz Batman’ni yoqtirasiz — mana misol” inline label that names the interest being used.
— A returning-user home that names prior state (“Oxirgi safar EKUKda to’xtagansiz”).

Replace the hardcoded "density" return in weakest.ts:37.
Route by actual subject of the bad mark, and surface the reason text on the Bugungi star: “Kecha fizikada 3 — Paskalni birga ko’rib chiqaylik.” Now the demo can open the diary, get a 3, and watch the badge move to a different topic with a different reason line. This single fix sells the personalization story.
Effort · half a day

Per-topic mastery integer in Firestore.
Store users/{uid}/mastery/{topicId} as a simple correct/total integer. Render the percentage on every StarTile; let litCount become “27% of constellation mastered.” Every star now has a number that visibly changes — the single highest-density “this app knows me” signal in EdTech.
Effort · 1–2 days

“Bu darsda misollar Batman uslubida” — read from profile.subInterests and inject the same string into the LLM prompt deterministically so the rendered example matches the chip. Personalization stops being invisible LLM mood and becomes a labeled, falsifiable UI promise.

Feedback · What happens after the constellation map?

Give the journey an end.

Muxammadali · verbatim

The 4-layer engagement loop · how every winner does it

Successful EdTech apps stack four loops at different time scales: immediate (cards, XP tick), daily (streak, daily quest), weekly (league, leaderboard), and lifelong (badges, certificates, public titles). Khan goes Meteorite → Black Hole over 10M energy points. Duolingo’s “5 years on Duo” anniversary makes the veteran legible to a stranger. Prodigy has a pet roster other kids can see. The 5-year-veteran looks different.

Lifelong · the “done” card is a flat string

The done card type at lib/lesson.ts is literally {title, body}. After all 6 stars in a constellation ignite: nothing happens. No chapter ceremony. No supernova. No certificate. No permanent artifact. No profile change. Exactly the void the mentor named.

The brand already has the long-arc metaphor sitting unused

Use it. Five payoffs that extend the star metaphor and lean into Uzbek astronomy heritage (Ulug’bek’s Samarqand observatory, Beruniy’s celestial measurements):

— Supernova on chapter completion. The 6th star ignites → the constellation collapses into a single named supernova labeled with the student’s name + date (“Murodjonning supernovasi · 2026-yil noyabr”). Permanent. 5-second cinematic. Replaces today’s nothing-happens.
— The Ulug’bek Catalog (Zij-i Sultani 2.0). Every supernova gets a row in the student’s personal zij — modeled on the 1437 Samarqand catalog of 1,018 stars. After a year, they have their own zij. Exportable PDF. Shareable link.
— Galaxies from constellations. Six supernovas (one per subject-unit) merge into a galaxy on the profile. The 5-year-veteran has multiple galaxies; the new student has a few ignited stars. Legibility-at-a-glance.
— Beruniy / Ibn Sino / Ulug’bek tiers. Three milestone titles tied to depth: Beruniy darajasi (3 chapters), Ibn Sino darajasi (6 chapters across 2 subjects), Ulug’bek darajasi (12 chapters — the year-veteran). Each visibly changes the night-sky background.
— Mahalla guilds. Students join a mahalla (school, class, friends). Weekly: whose mahalla ignited the most stars. Replaces Duolingo’s cold leagues with a culturally native unit kids already understand.

Feedback · Students should pay, not parents

The one place Muxammadali is wrong.

Muxammadali · verbatim

“ota onalardan ko’ra b2c da o’quvchilar pul to’lashi uchun ko’proq urgu beringlar dedi.”

Who actually pays in EdTech B2C

The pattern is clean: self-pay works when the product is identity, status, or urgent utility. Parent-pay works when the product is outcomes, safety, or prestige investment. Brilliant ($27.99/mo) is ~70% self-paying adults — sold as “stay sharp” status, not homework help. Photomath is mostly students themselves — pure utility, homework due in 6 hours, paid because the alternative is failing. Quizlet ($35.99/yr) is student-pay because it’s cheap enough for a teen’s pocket money. Chegg / CourseHero are 100% student-pay because the user is 18–24 with their own credit card.

The Uzbekistan reality check

Realistic parent-payer share at 80K/mo

The parent-pay budget is already provisioned and flowing — often the single largest discretionary spend after food. Education is non-negotiable parental duty in Central Asian family structure. Repetitor (tutor) culture is prestige for parents. Teens have Spotify-tier pocket money — not serious-EdTech-subscription money. Almost no teen-owned cards under 18. Culturally, a 14-year-old buying their own learning is abnormal — parents would feel shamed.

At 80K so’m, realistic distribution: 85% parent · 10% older-sibling/relative · 5% self-pay 10–11 sinf preppers. Switching to student-pay-first messaging at this price destroys it — teens won’t, parents will feel the product isn’t for them.

The strategic call

Muxammadali is solving a real problem: the student is the user. But he’s conflating user with buyer. The student is the user; the parent is the buyer; the product serves both, but the deck speaks to the buyer. This is exactly how Kumon, Khan Academy Kids, and every Korean hagwon app positions. The same product surface can feel like “yours” to the student while the payment flow feels like “your investment” to the parent.

Currently the profile only models the student. One day’s work unlocks the buyer-facing surface and enables targeted daily briefs.

Ship the daily Uzbek Telegram brief to parents.
“Bugun Aziz Arximed qonunini o’rgandi. Kechki ovqatda so’rang: ‘suvga tushgan tosh nega ko’tarilmaydi?’” Start manual (you sending 10 parents yourself) — this is the wedge that justifies the 80K price. Becomes the retention weapon.
3
Strengthen pitch slide 6 with the parent anchor.
“Repetitor 200K/soat. Scorpius 80K/oy — cheksiz. Ota-ona ko’radi, bola o’rganadi.” Add the 41–65% repetitor-pay anchor. Don’t apologize for parent-pays — it’s the strongest unit economics story on offer.
Optional hedge

A 25K so’m/oy “O’quvchi” tier with Humo AI only, no parent dashboard, for the 5–10% of self-paying 10–11 sinf students prepping for DTM. Keep it as a secondary line in the deck, not the headline. This catches Muxammadali’s edge case without diluting the main story.

YC-mentor verdict · pushback on Muxammadali
Reject the reframe. Tell him: user ≠ buyer. The budget you want already exists in the parent’s wallet (41–65% of surveyed families pay 500K+ for a tutor). Student-pay collapses LTV from 1.9M to <500K and CAC stops penciling. Two surfaces, one subscription — that’s the answer. Send him this section if there’s disagreement — the math is on this side.

Feedback · Looks like a test with prettier design

Build the Burj.

Muxammadali · verbatim

“iloji bo’lsa yoshroq o’quvchilarga o’yinga o’xshatib xar hil jarayonlar qo’shingalr sizlarniki xozir bir xil testga o’xshan qolgan faqat test dizayni yaxshiroq xolda ekan, goya yaxshi lekin…”

Why he’s right · what reads as “test”

Look at LessonDeck.tsx. The five quiz-grammar primitives are all there: (1) progress-dot rail at the top — kids read this as “5 questions remaining,” (2) the literal “Davom etish uchun javob bering” lock pill — the single most quiz-like artifact in the app, (3) A/B/C/D button rows with green check + red shake + explain reveal — the visual grammar of every standardized testing UI since 2010, (4) no persistent state between lessons — finish a lesson, blank slate next, (5) no surprise — every completion produces the same explain box.

What’s already game-shaped (keep): simulation, explore-sandbox, challenge, pattern-discover, compare-and-decide, build. DivisorGridSim lets the learner discover factor pairs by dragging a slider — that’s a DragonBox moment. The TikTok-style vertical paging is the right gesture; it’s the content inside that reads as quiz.

The 5 patterns that turn “test” into “game” at 10–14yo

(A) Loop-hiding via meta-progression. Prodigy never says “you’re doing math” — the kid says “I’m farming the Dark Tower.”
(B) Collectible identity. Avatars, pets, gear. Erikson identity formation at 11–12yo. Critical: avatar must visibly change after a session.
(C) Variable-ratio reward. Duolingo’s chest. 50% chance of gems after a lesson. Variable beats fixed at this age.
(D) Agency that isn’t “right answer.” Pick a class, pick a path, pick a 2-of-4 challenge. 11–12yo are leaving obedience stage.
(E) Social proof. Class leaderboard, mahalla guild, sinfdosh competition.
